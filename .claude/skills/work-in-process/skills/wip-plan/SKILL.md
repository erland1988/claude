# wip-plan

为指定模块生成详细执行计划，**按依赖顺序分阶段拆解**。

## 执行流程

1. 读取模块设计文档
2. **按层次分阶段拆解**：
   - Phase 1: 数据层（DDL/SQL/迁移脚本）
   - Phase 2: 模型/实体（数据结构定义）
   - Phase 3: 业务逻辑（核心业务实现）
   - Phase 4: 接口/入口（API/CLI/UI）
   - Phase 5: 联调验证（集成测试）
3. 每阶段拆分为**可执行步骤**（2-5分钟/步骤）
4. 输出执行计划 → `.wip/{name}/modules/{module}/plan.md`
5. **自动更新 ledger.md**

## 核心产出要求

1. **按层次分阶段**：数据层 → 模型/实体 → 业务逻辑 → 接口/入口 → 联调验证
2. **步骤统一模板**：`文件` / `位置` / `背景` / `操作` / `验证`
3. **测试矩阵**：`场景 | 前置条件 | 操作步骤 | 预期结果`
4. **步骤号管理**：一经分配不复用；取消步骤保留占位
