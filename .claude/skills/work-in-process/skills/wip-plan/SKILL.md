# wip-plan

为指定模块生成详细执行计划，**按依赖顺序分阶段拆解**。

## 执行流程

1. 读取模块设计文档 `.wip/{project}/modules/{module}/design.md`（wip-build 产出）
2. **按层次分阶段拆解**：
   - Phase 1: 数据层（DDL/SQL/迁移脚本）
   - Phase 2: 模型/实体（数据结构定义）
   - Phase 3: 业务逻辑（核心业务实现）
   - Phase 4: 接口/入口（API/CLI/UI）
   - Phase 5: 联调验证（集成测试）
3. 每阶段拆分为**可执行步骤**（2-5分钟/步骤）
4. **步骤统一模板**（每个步骤缺一不可）：
   - **文件**：目标文件路径，标注新建/修改
   - **位置**：精确到方法名 + 行号锚点
   - **背景**：为何改（仅在原因不直观时需要）
   - **操作**：具体代码/SQL，可直接落地
   - **验证**：编译/单测/联调的具体方式
5. **联调验证阶段**用测试矩阵呈现：`# | 场景 | 前置条件 | 操作步骤 | 预期结果`，覆盖正常流、边界、幂等、并发
6. 输出执行计划 → `.wip/{project}/modules/{module}/plan.md`
7. **自动更新 ledger.md**

## 核心产出要求

1. **按层次分阶段**：数据层 → 模型/实体 → 业务逻辑 → 接口/入口 → 联调验证
2. **步骤统一模板**：`文件` / `位置` / `背景` / `操作` / `验证`
3. **测试矩阵**：`场景 | 前置条件 | 操作步骤 | 预期结果`
4. **步骤号管理**：一经分配不复用；取消步骤保留占位

## 步骤号管理规则

- 步骤号一经分配不复用
- 迭代中被取消的步骤保留占位，注明「已移除 + 原因」
- 不静默删除，保持可追溯
